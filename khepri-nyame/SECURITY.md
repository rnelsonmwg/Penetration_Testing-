# Security and Responsible Use

Khepri Nyame is for authorized testing only. Use it only against assets you own, manage, or have explicit written permission to test.

## First-version safety controls

- Local-first execution
- Local JSON storage
- Authorization statement required at engagement creation
- Active testing approval model included but passive-only by implementation default
- Secret evidence redaction
- External tool wrappers default to dry-run
- Blocked unsafe workflow terms

## Prohibited usage

Do not use Khepri Nyame to perform unauthorized access, credential attacks, brute force, stealth scanning, persistence, destructive testing, data extraction, or exploitation.

## Reporting security issues in the tool

Open a GitHub issue or private advisory in your own repository if you extend this scaffold into a public project.
